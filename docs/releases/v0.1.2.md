# Night Voyager v0.1.2

> Release type: `local synthetic portfolio release`

## Summary

Night Voyager `v0.1.2` 发布完整 Governed Collaboration Core v1。受治理的共享会话把 `MessageEvent` communication、typed `MemoryCandidate` proposal、advisor verification、versioned `ConfirmedFact` publication 与后续 Case revision 串成一条可追溯 authority path；versioned Skill catalog、evaluation、activation/rollback 和 runtime task/execution pins 同时进入 current release。Primary `/demo` 继续承载 advisor-to-family walkthrough，secondary `/demo/collaboration` 提供 task-free browser walkthrough 与 read-only Planning Skill inspector。

## Completion

- Governed memory authority 固定为 `MessageEvent -> MemoryCandidate -> advisor verification -> ConfirmedFact -> Case revision`；消息和 proposal 不能自行获得 Case authority。
- assigned advisor 通过 deterministic atomic gate 确认或拒绝 revision-pinned candidate；确认后发布 immutable versioned fact 并推进 Case revision，不自动创建 `AgentTask`。
- versioned Skill governance 提供六个 product-owned Skill definition、deterministic evaluation、owner activation/rollback，以及新 task 和 execution 的 exact runtime pins；packaged runtime compatibility 在 adapter start 前 fail closed。
- secondary `/demo/collaboration` 证明 parent proposal、advisor confirmation 与 authoritative fact/revision reload，保持 local synthetic、task-free，不改变 primary `/demo` 的 advisor-to-family product story。
- read-only Planning Skill inspector 展示 server-projected active version、evaluation provenance 与 task pin status；浏览器不能激活、回滚或改变 runtime authority。
- `v0.1.1` 以来合并的 routine dependency and repository maintenance 包括 React / ReactDOM `19.2.7`、pytest `9.1.1`、Ruff `>=0.15.22`、Node / React types 与 ESLint patch updates、PostgreSQL Alpine variant/digest pin，以及 GitHub Actions setup updates；这些 maintenance facts 不改变 Governed Collaboration Core v1 的产品主线。本次 release-prep diff 未再引入额外 dependency upgrade、migration、API 或 runtime behavior 变更。

## Verification

Release candidate 必须依次通过 focused release/documentation tests、`uv lock --check`、`make doctor MODE=dev`、`make collaboration-check`、`make skills-check`、`make db-check`、`make check`、`make proof`、`make compose-proof`、`make down`、最终为空的 `docker compose ps --all`、clean-tree `uv run python scripts/verify_release.py --tree-mode release`、Ruff、Pyright、frontend lint/typecheck/Vitest/production build、`git diff --check`、historical release digest verification 与 public-hygiene review。

Live provider proof was not run；它不属于 required CI、offline proof 或本次 release gate，仍需单独授权。

## Scope

本版本的发布 artifact 是 GitHub-generated source archive。范围限定为已经落地的 Governed Collaboration Core v1、secondary local synthetic browser walkthrough、read-only inspector、`v0.1.1` 以来已合并的 routine maintenance，以及 current-release identity/documentation 收口。本次 release-prep diff 不新增额外 dependency upgrade，也不重新设计 CI。Live DRA/MKE product integration、production deployment、真实学生、production tenancy、SLA、业务收益、tag、GitHub Release 和 public archive publication 均不属于 release-candidate 实施范围。

## Risk / Impact

本版本证明有界、可复现的 local synthetic collaboration、memory 与 Skill governance closure，不证明 live provider 可用性、production tenancy、真实机构覆盖、录取结果、SLA、性能收益或业务影响。`/demo/collaboration` 是 secondary、task-free walkthrough；`/demo` 仍是 primary advisor-to-family walkthrough。未运行 live provider proof。
