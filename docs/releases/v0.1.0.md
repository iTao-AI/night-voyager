# Night Voyager v0.1.0

> Release type: `local synthetic portfolio release`

## Summary

Night Voyager `v0.1.0` 交付一条完整的 local synthetic 顾问到家庭决策流程。可追溯 synthetic Evidence 与确定性 policy 生成路线比较；顾问决定哪些内容可以进入家庭讨论；家庭确认 server-derived 预算边界后，系统持久化 Decision Receipt 与 Timeline Plan。

## Completion

- 通过 transport-only BFF 将 Next.js `/demo` 接入 role-scoped FastAPI read/mutation endpoints 与 PostgreSQL authority。
- 保留 forced RLS、runtime-role separation、窄 database functions、idempotency、advisor/family human gates 与 non-enumerating authorization failures。
- 使用 PostgreSQL 实现 durable AgentTask，覆盖 bounded lease、heartbeat、generation fencing、retry/cancel/reclaim、worker restart durability 与 authorized SSE replay/reconnect。
- M4B 仍是 optional local read-only MKE compatibility proof；所有 MKE 投影保持 `UNTRUSTED_CANDIDATE`，默认产品路径保持 MKE-free。
- 修正 CNY minor-unit 展示，并将已批准的 primary UI domain codes替换为 closed human-readable copy。

## Verification

Release candidate 必须通过 `make doctor MODE=dev`、`make check`、`make proof`、`make compose-proof`、1440/768/390 real Chromium、public-hygiene 与 clean diff review。只有 reviewed tree 与 squash-merged `main` tree一致、hosted required checks成功后，才发布正式 GitHub Release。

发布后从全新临时目录解压 public source archive，并运行 `make doctor`、`make proof`、`make compose-proof`、`make down`；最终 `docker compose ps --all` 必须为空。

## Scope

本版本的完整 release artifact 是 GitHub-generated source archive。PyPI package、container image、custom wheel asset、deployment、domain、hosted SaaS 与 remote provider均不属于本版本。

## Risk / Impact

本版本只证明可复现的 local synthetic 工程路径，不证明 production tenancy、live institutional coverage、真实学生、录取结果、distributed high availability、SLA、性能目标或业务收益。

## Documentation impact

Bilingual README、connected-demo runbook、architecture/reference material、screenshots、release notes 与 source-archive verification procedure 共同描述同一个有界的 `v0.1.0` 行为。
