# Night Voyager v0.1.3

> Release type: `local synthetic portfolio release`

## Summary

Night Voyager `v0.1.3` 发布 `v0.1.2` 之后已经合并的 governed fact-to-plan closure、Chinese-first bilingual presentation、High-End Portfolio Entry 与 Next.js security patch。完整 local synthetic walkthrough 将同一个 same Case 从 `/demo/collaboration` 的 confirmed facts 带入 `/demo`，且只有用户显式操作才启动规划。authority path 保持为 `Browser -> Next.js BFF -> FastAPI -> PostgreSQL -> worker/SSE`，再进入 advisor review、family decision、receipt 与 timeline。

## Completion

- migration `0009` 把 first deterministic task creation 固定为 explicit planning-start authority；Case transition、five-field Skill pin、`AgentTask`、dispatch、first event 与 idempotency response 在一个 atomic PostgreSQL transaction 中写入，legacy transition authority 已从 runtime role 收回。
- governed walkthrough 在 handoff 时只做 same-Case authority reload；handoff 不创建 task。用户在 `/demo` 的显式操作才创建规划任务并继续 durable worker/SSE、advisor 与 family closure。
- root、`/demo` 与 `/demo/collaboration` 默认 exact `zh-CN`，只接受显式 exact `en` switch。locale 仅属于 presentation，不进入 session、HTTP/BFF、task、SSE 或 domain authority。
- “虚幻夜航”入口使用 responsive AVIF / WebP。首页预算展示 `30–40 万元` / `CNY 300,000–400,000`；governed detailed Evidence 保留 exact `305,500–400,000 CNY`。
- Next.js `16.2.11` 与 eslint-config-next `16.2.11` 关闭该官方版本列出的 direct advisories。`sharp@0.34.5` 的 `GHSA-f88m-g3jw-g9cj` 仍是 optional/transitive deferred risk；本版本不声称 audit-zero，也不加入 unsupported `sharp@0.35.x` override。

## Verification

Release candidate 必须通过 focused release/documentation RED -> GREEN、`make doctor MODE=dev`、`uv lock --check`、Ruff、Pyright、frontend lint/typecheck/Vitest/production build、`make check`、`make proof`、单次 top-level `make compose-proof`、`make down`、空的 `docker compose ps --all`、clean-tree release verifier、diff/public-hygiene/private-path/secret scans，以及六份 historical document SHA-256 contract。

Live provider proof was not run；deploy、tag、GitHub Release 与 public archive Gate E 也不属于本地 release-prep。

## Scope

本 patch release 只发布已合并能力，并把 package identity、release contract 与 current documentation 收敛到 `v0.1.3`。发布 artifact 仍是 GitHub-generated source archive。DRA 仍是 deterministic offline governed candidate import；MKE 仍是 optional read-only untrusted candidate boundary。release-prep does not change migration, API, runtime behavior, or dependency tree。

## Risk / Impact

本版本是 local synthetic portfolio release：no production deployment，no real students、真实学校覆盖、顾问团队或录取结果，也没有 live DRA/provider product integration。worker/SSE 是 local durable proof，not distributed HA or SLA。它不证明 production tenancy、availability、业务收益或 admissions outcome。

`sharp@0.34.5` 的 `GHSA-f88m-g3jw-g9cj` 保持明确 deferred；因此不得把本版本描述为 audit-zero 或所有依赖风险已关闭。

## Documentation impact

Bilingual README、documentation index、contributor/security entry、current design/status banner、ADR/spec/plan status、release verifier、architecture/unit contracts、release notes 与 verification guide 现在以 `v0.1.3` 为 current release。`v0.1.0`、`v0.1.1` 与 `v0.1.2` 的六份 published documents 保持 byte-identical immutable history。
